# 递归

递归

递归是一种解决问题的方法，它解决问题的各个部分，

直到解决最初的大的问题。通常涉及到函数自身的调用。

// 递归
function getMaxStack() {
  let i = 0;

  function recursive() {
    i++;

    recursive();
  }

  try {
    recursive();
  } catch (error) {
    return i;
  }

  return i;
}

// 实例调用
let maxTimer = getMaxStack();
console.log( maxTimer );


// 返回的最终结果 和操作系统和浏览器的不同，具体数值会有所不同，但是区别相差不是很大。

ES6 尾部调用优化

如果在函数中的最后一个操作是调用函数，会通过"跳转指令" 而不是"子程序调用"来控制。

也就是说，在ES6中，recursive 函数会一直执行下去。所以必须有停止递归的边缘条件。

斐波那契数列

  1. 1 和 2 的斐波那契数是1

  2. n (n > 2) 的斐波那契数是n - 1的斐波那契数加上n - 2的斐波那契数

// 递归形式实现
function fabonacci(num) {
  if (num === 1 || num === 2) {
    return 1;
  }

  return fabonacci(num - 1) + fabonacci(num - 2);
}

// 非递归形式出现
function fib(num) {
  let n1 = 1;
  let n2 = 1;
  let n = 1;

  for (let i = 3; i <= num; i++) {
    n = n1 + n2;
    n1 = n2;
    n2 = n;
  }

  return n;
}

ES6中的递归

ES6中因为尾部调用优化的缘故，递归并不会慢，但是其他语言中，递归是非常慢的。
